## Summary

Link or unlink a header/footer slot to/from the previous section.

- Operation ID: `headerFooters.refs.setLinkedToPrevious`
- API member path: `editor.doc.headerFooters.refs.setLinkedToPrevious(...)`
- Mutates document: `yes`
- Idempotency: `conditional`
- Supports tracked mode: `no`
- Supports dry run: `yes`
- Deterministic target resolution: `yes`

## Expected result

Returns a SectionMutationResult receipt; reports NO_OP if the link state already matches, INVALID_TARGET for the first section.
